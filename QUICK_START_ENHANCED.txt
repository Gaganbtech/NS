📊 KEY METRICS EXPLAINED
═══════════════════════════════════════════════════════════════════════════════

EXECUTION TIME
  What: Time from command start to completion
  RSA:    ~1.2 seconds (fastest)
  PQC:    ~1.5 seconds (moderate)
  Hybrid: ~1.7 seconds (slowest)
  Why: Larger keys = more processing

PACKET SIZE
  What: Total bytes in OpenSSL output
  RSA:    ~3,000 bytes (smallest)
  PQC:    ~4,500 bytes (larger)
  Hybrid: ~5,500 bytes (largest)
  Why: PQC keys are 1184 bytes vs RSA 256 bytes

SECURITY LEVEL
  RSA:    Medium ⚠️  (vulnerable to quantum)
  PQC:    High ✅    (quantum-resistant)
  Hybrid: Very High 🔥 (double protection)

🔍 WIRESHARK INTEGRATION
═══════════════════════════════════════════════════════════════════════════════

1. Start Wireshark:     sudo wireshark
2. Apply Filter:        tls or tcp.port == 4433
3. Run Test:            Click RSA/PQC/Hybrid button
4. Analyze Packets:     Client Hello, Server Hello, Key Exchange
5. Compare Sizes:       RSA < PQC < Hybrid
6. Verify Key Exchange: X25519 vs MLKEM768 vs X25519MLKEM768
