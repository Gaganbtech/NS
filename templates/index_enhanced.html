🔍 Wireshark Proof Guide

How to Capture and Analyze TLS Traffic

Step 1: Start Wireshark

sudo wireshark

Launch Wireshark with administrator privileges

Step 2: Select Network Interface

Choose the interface where traffic flows (e.g., lo for localhost, eth0 for network)

Step 3: Apply TLS Filter

tls

Or for specific port:

tcp.port == 4433

Step 4: Run TLS Test

Click RSA, PQC, or Hybrid button in this dashboard

Step 5: Analyze Packets

  • Client Hello: Check supported cipher suites and groups
  • Server Hello: See selected cipher and key exchange
  • Packet Length: Compare RSA vs PQC vs Hybrid sizes
  • Key Exchange: Look for MLKEM768 or X25519MLKEM768

Step 6: Compare Results

Mode Expected Key Exchange Packet Size
RSA X25519 or RSA ~2-3 KB
PQC MLKEM768 ~4-5 KB (larger)
Hybrid X25519MLKEM768 ~5-6 KB (largest)
Note: PQC and Hybrid modes require OQS-enabled OpenSSL. Install using: ./scripts/setup_oqs.sh
